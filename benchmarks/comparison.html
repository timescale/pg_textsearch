pg_textsearch vs System X

Development Progress Report

Snapshot in time: This comparison reflects pg_textsearch as of February 6, 2026. The project is under active development with performance improvements shipping regularly. Check the benchmark dashboard for the latest numbers.

Dataset: MS MARCO 8.8M passages | Date: 2026-02-06 | Commit: 07b8656 | System X: v0.20.6

Current Status

pg_textsearch today

  • 1.8x faster overall query throughput
  • Smaller index (no positions stored)*
  • Faster on most queries (1-7 tokens)
  • Parallel index build (4 workers)
  • Native Postgres integration

System X v0.20.6

  • Faster index build (2x)
  • Faster on 8+ token queries
  • Phrase queries supported
  • Larger feature set (facets, etc.)

Recent Improvements

  • Parallel index build - Now uses 4 workers, cutting build time in half (PR #188)
  • Query performance - 3-9x faster on short queries vs previous release
  • Overall throughput - pg_textsearch now 1.8x faster than System X

Analysis

Short and medium queries (1-7 tokens): pg_textsearch wins

pg_textsearch now wins on 7 out of 8 token buckets, with dramatic improvements on short queries (3-9x faster). The Block-Max WAND implementation with compressed posting lists excels at pruning non-competitive documents early.

Long queries (8+ tokens): System X leads

System X maintains a lead on the longest queries (8+ tokens), where it's 18% faster at p50 and 66% faster at p95. This is an active optimization target for pg_textsearch.

Overall throughput: pg_textsearch wins

With the improvements to short and medium query performance, pg_textsearch now delivers 1.8x faster overall throughput than System X on the MS-MARCO benchmark (13.3s vs 23.6s for 800 queries).

Methodology

Both extensions benchmarked on identical GitHub Actions runners with the same Postgres configuration. See full methodology for details.

Note: Both extensions use default configurations. Results are from a single run; expect ~10% variance between runs. This page will be updated as optimizations land.
